# /ghost

## Purpose
Generate or rewrite text in the user's voice and style by reading their existing wiki notes as voice samples.

## Workflow
1. Take a draft or topic as input.
2. Sample 20+ wiki pages the user has touched (prefer comparison and synthesis pages, plus any concept pages updated by the user directly).
3. Identify recurring stylistic patterns: sentence length, vocabulary, hedging language, structural habits, formatting conventions.
4. Generate or rewrite the requested text in that style.
5. Briefly note which patterns informed the rewrite so the user can confirm or correct.

## When to Use
When drafting longer-form content (essays, summaries, posts) where the user wants their own voice preserved rather than a generic LLM tone.
